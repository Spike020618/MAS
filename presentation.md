## 📐 核心公式与理论基础

### 三个核心公式

| 公式 | 含义 | 作用 |
|------|------|------|
| **公式1**: `E = (1/2) Σ w_ij (θ_i - θ_j)²` | 共识能量 | 衡量全网节点参数一致性，目标是E→0 |
| **公式2**: `min_θ [-(ΣU_i) + λ·E]` | Leader优化目标 | λ权衡激励与一致性 |
| **公式3**: `θ_i^{t+1} = θ_i^t + η∇U_i - γΣ_j w_ij(θ_i-θ_j)` | 分布式学习 | 无中央控制的参数更新 |

### 关键洞察

- **博弈论视角**: Leader设置激励参数θ，Followers响应优化自身效用
- **分布式视角**: 节点间P2P通信，无单点故障
- **共识视角**: 语义相似度驱动参数收敛

## 🎮 Stackelberg Consensus Game (SCG) 详解

### 博弈框架

**Stackelberg领导者-跟随者博弈** + **共识约束**

- **领导者 (Leader)**: 平台协调者，设置激励参数θ_i
- **跟随者 (Followers)**: 各智能体节点，根据θ_i优化自身策略
- **共识约束**: 通过能量函数E保持参数一致性

### 博弈流程

```python
# 1. 初始化激励参数
θ_i ∈ [0.5, 1.0]  # 随机初始化

# 2. 跟随者响应（每轮）
for each agent i:
    U_i = quality_score * θ_i - cost_i  # 效用计算
    strategy_i = argmax(U_i)            # 最优策略选择

# 3. 领导者更新（公式2）
min_θ [-(ΣU_i) + λ·E]  # 梯度下降优化

# 4. 共识约束（公式3）
θ_i^{t+1} = θ_i^t + η∇U_i - γΣ_j w_ij(θ_i - θ_j)

# 5. 收敛判断
while E > ε and round < max_rounds:
    continue
```

### 参数调优策略

| 参数 | 含义 | 推荐值 | 影响 |
|------|------|--------|------|
| **λ (lambda_c)** | 共识-激励权衡 | 1.0 | λ=0纯博弈，λ=1平衡，λ>1强共识 |
| **η (eta)** | 学习率 | 0.05 | 控制更新步长 |
| **γ (gamma)** | 共识强度 | 0.1 | 参数一致性压力 |

### 收敛验证实验

```
λ=1.0 收敛曲线示例：
Round  1: E=0.1423  ΔΘ=0.0891
Round  5: E=0.0614  ΔΘ=0.0312
Round 10: E=0.0128  ΔΘ=0.0047  ← 收敛点
Round 15: E=0.0031  ΔΘ=0.0009  ✅ 单调递减验证
```

## 🔗 智能体共识机制

### AEIC 数据结构

每个节点生成结构化决策记录：

```json
{
  "node_id": "node_1",
  "assumptions": "申请人信用评分良好，收入稳定",
  "evidence": ["征信报告", "银行流水", "收入证明"],
  "inference": "各项指标满足准入条件，无不良记录",
  "conclusion": "批准贷款申请"
}
```

### 多节点共识评估

**核心创新**: 支持N节点全量评估，而非仅两两对比

```python
# 主接口
result = consensus_engine.evaluate_consensus(node_records)

# 返回结果
{
  "avg_similarity": 0.87,           # 全网平均相似度
  "pairwise": [...],                # 所有节点对相似度
  "similarity_matrix": [[...]],     # N×N相似度矩阵
  "utility": [...],                 # 各节点效用向量
  "decision": "ESS_Consensus"       # 共识决策
}
```

### 相似度计算方法

| 方法 | 速度 | 精度 | 适用场景 |
|------|------|------|----------|
| `char_jaccard` | ⚡⚡⚡ | ☆ | 基准测试 |
| `word_tfidf` | ⚡⚡ | ★★★ | 大规模离线 |
| `bm25` | ⚡⚡ | ★★★★ | **默认推荐** |
| `sentence_bert` | ⚡ | ★★★★★ | 高精度需求 |
| `llm_judge` | 🐢 | ★★★★★ | 最高精度评测 |

### 加权相似度融合

```
sim_total = 0.2·sim(A) + 0.3·sim(E) + 0.2·sim(I) + 0.3·sim(C)
```

其中：
- A = Assumptions (前提假设)
- E = Evidence (支撑证据)  
- I = Inference (推理过程)
- C = Conclusion (最终结论)

### 共识决策逻辑

| 相似度范围 | 决策结果 | 含义 |
|------------|----------|------|
| > 0.8 | ESS_Consensus | 全网高度一致，可直接采纳 |
| 0.6-0.8 | Audit_Required | 需要人工审核 |
| < 0.6 | Reject | 存在重大分歧，建议重做 |

## 🧪 实验验证与结果

### 三大核心实验

#### Exp-1: 收敛性验证
- **目标**: 验证公式3的收敛性
- **方法**: 多λ值迭代，观察E单调递减
- **结果**: 所有λ值下均实现收敛，E从0.14降至0.003

#### Exp-2: 语义方法对比
- **目标**: 评估5种相似度方法的性能
- **指标**: MAE、三分类准确率、ESS率、平均效用
- **发现**: BM25在速度-精度平衡中表现最佳

#### Exp-3: 基线对比
- **目标**: 验证SCG方法的优越性
- **对比方案**: Random、Pure-Stackelberg、Pure-Consensus、SCG
- **结果**: SCG在社会福利上显著优于其他方案

### 关键发现

1. **博弈+共识的协同效应**: λ=1时社会福利提升23%
2. **分布式学习的稳定性**: 无中央控制仍能实现参数收敛
3. **语义共识的实用性**: BM25方法在实际应用中表现优秀
